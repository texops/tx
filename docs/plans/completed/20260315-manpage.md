# tx manpage

## Overview
Create a comprehensive manpage for the `tx` CLI in mdoc format at `man/tx.1`, and integrate it into the release pipeline so it is included in release archives. The Homebrew formula update (in `texops/homebrew-tap`) is tracked as a separate follow-up.

The manpage is intentionally a hybrid reference/tutorial document — it serves as the single comprehensive doc for tx, covering both quick-start guidance and full command reference.

## Context
- CLI commands defined in `internal/cli/commands.go`
- Config parsing in `internal/cli/config.go`
- Auth/credentials in `internal/cli/global_config.go`
- TTY/interactive detection and confirmation prompts in `internal/cli/ui.go`
- File collection and ignore rules in `internal/cli/hasher.go`
- Entry point, `version` and `--version` in `cmd/tx/main.go`
- GoReleaser config in `.goreleaser.yaml`
- Homebrew formula managed in external repo `texops/homebrew-tap`

## Development Approach
- No tests needed (documentation-only change)
- Verify manpage renders correctly with `mandoc`

## Implementation Steps

### Task 1: Write the manpage

**Files:**
- Create: `man/tx.1`

- [x] Write mdoc manpage with these sections in order:
  - NAME: `tx - TexOps CLI to build LaTeX documents in the cloud`
  - SYNOPSIS: all command forms (`login`, `init`, `build`, `status`, `token create/list/delete`, `version`, `--version`)
  - DESCRIPTION: one paragraph — what tx is, what it does, interactive + CI support
  - GETTING STARTED: sequential walkthrough (login → init → build), brief CI note
  - COMMANDS: each command with flags and behavioral notes
    - `login`: device code flow, opens browser
    - `init`: creates .texops.yaml, discovers .tex files; selection prompts (TexLive, compiler, documents) require both stdin and stdout to be a TTY (`IsInteractive()`); flags `--texlive`, `--compiler` for non-interactive use
    - `build`: positional doc args, `--no-cache`, `--yes` (suppresses large-upload confirmation prompt; the prompt is shown when the combined size of files to upload exceeds 50 MB; when stdout is not a TTY, confirmations default to yes); builds all documents if no args; if `.texops.yaml` is missing and stdout is a TTY (`IsTTY()`), offers to run init first
    - `status`: shows email (when available), auth method, token expiry
    - `token create`: `--name`, `--expires-in`, `--no-expiry`
    - `token list`
    - `token delete`: positional name argument, or interactive selection if no argument given
  - CONFIGURATION: .texops.yaml format
    - Note that `tx init` generates a minimal config (project_key, texlive, compiler, documents with name and main); additional keys described below are optional and can be added manually
    - Top-level: project_key, texlive, compiler, api_url
    - documents list: name, main, directory, output, per-document overrides (compiler, texlive)
    - Valid compilers: pdflatex, xelatex, lualatex, latex, platex, uplatex
    - Validation rules:
      - `texlive` is required, must be a non-empty string (not validated against a fixed set of versions)
      - `documents` is required, must be a non-empty list of mappings
      - each document requires a unique `name` and non-empty `main`
      - `main`, `directory`, and `output` must not escape the project root
      - `output` defaults to `main` with `.pdf` extension; when `directory` is set, the effective default output path is relative to that directory
      - document-level `compiler` and `texlive` override top-level values
      - top-level `compiler` defaults to `pdflatex` if omitted
      - old-style configs with top-level `main` key are rejected with a migration hint
  - AUTHENTICATION:
    - Credential lookup order: TX_API_TOKEN → credentials file → keyring
    - Credential storage: `tx login` writes to keyring first; falls back to credentials file if keyring storage fails
    - Device flow, token management for CI
  - FILE HANDLING:
    - `.git/`, `.texops.yaml`, and `.txignore` are always excluded from uploads
    - `.gitignore` patterns are loaded from the project root
    - `.txignore` files can exist in nested directories with local scope; negation patterns supported
    - only files the server reports as missing are uploaded (hash-based comparison)
  - ENVIRONMENT: TX_API_TOKEN, TX_API_URL, XDG_CONFIG_HOME (affects credentials file location)
  - FILES: .texops.yaml, .txignore, credentials file at `$XDG_CONFIG_HOME/texops/credentials.yaml` (when `XDG_CONFIG_HOME` is set to an absolute path) or `$HOME/.config/texops/credentials.yaml` (otherwise)
  - EXIT STATUS: 0 (success, including help/version output), 1 (any failure: parse, config, auth, build, or runtime error)
  - EXAMPLES: build specific doc, --no-cache, CI pipeline snippet, multi-doc config example, token creation
